Review the current state of MYOH and Life Pilot and produce an implementation plan, gap analysis, and build update for the following missing capability areas.

Context
- MYOH already has Settings, Data, Defaults, and Generate Import wired in.
- ImportGenerator is live in MYOH Settings.
- TypeScript is compiling cleanly.
- Current import/export supports:
  - incomeSources
  - incomeEntries
  - bills
  - arrearsItems
  - tasks
  - commsEntries
  - weeklyEntries
  - gigEntries
  - budgetCategories
  - scenarios
- Household budgeting and rent arrears planning are already in progress.
- We do NOT want to break existing import/export behavior.
- We do want to extend the model cleanly so future imports, dashboards, and spending logic stay consistent.

What appears to be missing
1. BNPL does not appear to have a standalone import/generate-import layer.
2. Afterpay and other BNPL products need to be wired to:
   - payments
   - liabilities / outstanding balance
   - spendable or restricted purchasing capacity where relevant
3. Gift card / stored value needs to be treated as:
   - stored value
   - spendable balance
   - linked to purchases/payments where used
4. We likely need a forwarded-email intake mechanism so users can send:
   - agreements
   - receipts
   - bills
   - payment confirmations
   - arrears notices
   - screenshots/PDFs/documents
   into a controlled inbox for processing/import/review.
