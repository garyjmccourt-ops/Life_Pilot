API route:
Create artifacts/api-server/src/routes/gig-imports.ts with:

GET /api/gig-imports
- list staged imports
- default to pending first
- include approved/rejected if query param requests it

POST /api/gig-imports
- stages incoming Gig_Pilot records only
- accepts either one entry or a payload with entries[]
- validates with Zod
- requires source_system = "gig_pilot"
- requires source_ref
- validates person, platform, dates, amounts, payment_status
- warns but does not reject if person is not Gary or Sam
- detects duplicate source_system + source_ref
- duplicate should not create a second pending import
- do not write to gig_entries
- do not write to income_entries

PATCH /api/gig-imports/:id/reject
- sets review_status = rejected
- saves rejection_reason
- does not delete the staged row

PATCH /api/gig-imports/:id/approve
- for this packet, do not promote to gig_entries yet
- return a clear "approval promotion not implemented in this packet" response
- leave the row pending or mark it only if no live table write is needed
- promotion will be a later explicit packet
