11. CALCULATION RULES
========================

Family budget:
- Use actual IncomeEntries where dateReceived falls inside the selected budget period.
- Include expected income separately as forecast only.
- Do not double-count gig income if it already created a linked income entry.
- Safe to spend = received income - unpaid essential bills due in period - planned essential category spend - arrears commitments - emergency buffer.
- Show warning if safe to spend is negative.
- Show expected shortfall if upcoming bills exceed expected income.

Bills:
- amountRemaining = amount - amountPaid.
- status:
  - unpaid if amountPaid = 0
  - part-paid if amountPaid > 0 and amountPaid < amount
  - paid if amountPaid >= amount
  - overdue if dueDate has passed and amountRemaining > 0

Gig work:
- netIncome = grossEarnings + tipsBonus - fees - fuelEstimate - otherExpenses
- If fastPayAmount is entered, use that as received income unless otherwise overridden.
- If weeklyDepositAmount is entered, use that as received income unless otherwise overridden.
- Allow manual override.

Scenarios:
- Base scenario starts from current actual budget.
- Scenario assumptions should not alter live data until user applies them.
- Compare base vs
