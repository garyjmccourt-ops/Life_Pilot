# Repository Language Standard

Repository-facing Life Pilot documents should use role-based wording rather than personal-name wording.

## Preferred roles

- **Product Owner**: product intent, scope, terminology and feature decisions.
- **Build Owner**: implementation approval, build safety and release/governance decisions.
- **Project Manager**: coordination, sequencing, timelines and delivery tracking.

## Approval wording

Preferred wording:

> Deeper build changes require explicit approval from the Product Owner / Build Owner before implementation.

Avoid personal-name approval wording in repository items unless there is a specific authorship, legal, audit or historical reason.

## Build Control instruction

Build Control should reference this standard when preparing Replit Pilot or Build GPT prompts.

Use role-based wording in prompts and product notes so repository items remain portable, professional and easier to hand over.
